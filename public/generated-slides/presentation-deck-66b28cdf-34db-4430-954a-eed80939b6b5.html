01 / 认识 Mastra

什么是 Mastra？

Mastra 是一个开源的 TypeScript Agent 开发框架，把构建 AI 应用需要的核心能力打包成统一的编程模型 —— 你不用从零拼接 LLM 调用、工具、状态和流程编排。

01Agent

智能体

封装 LLM + 指令 + 工具，让模型具备自主决策与执行能力。

02Tool

工具调用

让 Agent 能调用函数、API、数据库，把语言转化为行动。

03Workflow

工作流

用图结构编排多步骤任务，支持分支、重试与人工介入。

04Memory / RAG

记忆与知识

持久化对话上下文，接入向量检索增强回答准确性。

npm run dev

http://localhost:4111

05 / 编写代码

创建你的第一个 Agent

一个 Agent = 模型 + 指令（Instructions） + 可选工具集，用一个函数就能定义。

import { Agent } from '@mastra/core/agent';
import { openai } from '@ai-sdk/openai';

export const weatherAgent = new Agent({
  name: 'weather-agent',
  instructions: '你是一个天气助手，简洁准确地回答天气问题',
  model: openai('gpt-4o-mini'),
});
name 唯一标识instructions 定义人设model 选择模型

06 / 让 Agent 动手

用 Tool 赋予行动力

Tool 是一个带类型校验的函数，Agent 会自主判断何时调用它。

import { createTool } from '@mastra/core/tools';
import { z } from 'zod';

export const getWeather = createTool({
  id: 'get-weather',
  description: '查询指定城市天气',
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ context }) => {
    return fetchWeather(context.city);
  },
});
①

定义输入结构

用 zod 声明参数，Mastra 自动生成校验与类型提示。

②

编写 execute

真正的业务逻辑：调用 API、查数据库都可以。

③

挂载给 Agent

在 Agent 配置里加入 tools: { getWeather } 即可。

07 / 编排复杂任务

用 Workflow 串联多步骤

当任务需要多个阶段、分支判断或人工审核，用 Workflow 把步骤显式建模成图。

Step

原子步骤

每个 Step 是一个独立、可测试的执行单元，可以是调用 Agent，也可以是普通函数。

Control Flow

分支与并行

支持条件分支、并行执行、循环重试，处理真实业务里的复杂路径。

Suspend/Resume

人工介入

流程可在关键节点挂起，等待人工确认后再继续执行，适合审批类场景。

08 / 记忆与可观测性

让 Agent 记住上下文，也让你看清它在想什么

生产级 Agent 离不开状态管理和调试能力，这两点 Mastra 都内置支持。

Memory

对话记忆 & RAG

按会话持久化历史消息，可选接入向量数据库做语义检索，让回答基于你自己的知识库。

Observability

Tracing & Eval

Playground 中可视化每一次模型调用、工具调用链路，并支持编写评测用例衡量效果。
