核心概念速览
第一部分：框架认知与基础概念
🤖

Agent（智能体）

由模型、系统指令（instructions）和可选工具组成的核心执行单元。

🔧

Tools（工具）

基于函数与 Zod Schema 定义的能力扩展，让 Agent 能调用外部 API 或计算。

📊

Workflows（工作流）

用于编排多步骤、有状态任务的图结构。

🧠

Memory（记忆）

管理对话上下文与长期记忆，提升交互连贯性。

编写你的第一个 Agent
第二部分：环境搭建与代码实战
// src/index.ts
import { Agent } from '@mastra/core';

export const myAgent = new Agent({
  name: 'Helper',
  instructions: '你是一个乐于助人的 AI 助手。',
  model: 'openai/gpt-4o'
});

代码解析

  • 引入 Agent：从 @mastra/core 导入核心类。
  • name：定义 Agent 的唯一标识与基础角色名。
  • instructions：设定系统指令，约束 Agent 的行为与语气。
  • model：指定底层 LLM 模型（如 openai/gpt-4o）。
5 / 8 Mastra Framework
为 Agent 添加工具（Tools）
第二部分：环境搭建与代码实战
import { createTool } from '@mastra/core';
import { z } from 'zod';

const weatherTool = createTool({
  id: 'get-weather',
  description: '获取指定城市的天气',
  inputSchema: z.object({ city: z.string() }),
  execute: async ({ context }) => {
    return `天气晴朗，位于 ${context.city}`;
  }
});
⬇️ 挂载到 Agent
export const myAgent = new Agent({
  name: 'Helper',
  instructions: '你是一个天气助手。',
  model: 'openai/gpt-4o',
  tools: [weatherTool] // 绑定工具数组
});
