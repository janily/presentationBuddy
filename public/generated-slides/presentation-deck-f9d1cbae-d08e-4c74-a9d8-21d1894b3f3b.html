WHY MASTRA / ADVANTAGES

为什么选择 Mastra？重塑 TS AI 开发范式

TRADITIONAL
传统 AI 框架的工程痛点
  • Python 垄断痛点 构建 TS 架构项目仍需额外维护 Python 微服务，链路冗长
  • LangChain.js 过于冗重 历史包袱多，API 繁琐，多层抽象常常阻碍底层逻辑控制
  • 类型安全形同虚设 LLM 调用传参容易出错，缺乏开箱即用的运行时严格类型保障
MASTRA SOLUTION
Mastra 的现代化解决方案
  • 直接嵌入 Web 架构 直接嵌入 NestJS, Next.js 或 Express，无需启动复杂微服务
  • 端到端强类型校验 底层基于 Zod 实现编译期与运行时的参数双重校验，杜绝意外调用
  • 为 Agentic 量身定制 极简现代化 API，原生对 RAG、Tools 调度和工作流提供完美支持

ARCHITECTURE / FOUR PILLARS

Mastra 的核心四大支柱

Agents 智能体决策核心

具备独立 System Prompt、能够自主分析并选择注册工具（Tools）进行迭代推理的核心大脑单元。

Tools 强类型原子工具

使用 Zod 声明精准入参规格（Input Schema），可被 Agent 自主执行的任何 JavaScript / TypeScript 异步函数。

Workflows 确定性工作流

以图（Graph）或链式结构运作的流程控制器。用于在确定性业务流程中约束 AI 边界，规避幻觉失控。

MCP 标准多源集成机制

全面对接 Model Context Protocol (MCP) 标准，使 Agent 能即插即用集成外部文件系统、第三方 API 与 SaaS 生态。

结构化积木式架构 · 极佳上手体验

PRACTICAL 1 / FIRST AGENT

实战 1：编写你的第一个 AI Agent

src/mastra/agent.ts
import { Agent } from '@mastra/core'; import { openai } from '@ai-sdk/openai'; export const chefAgent = new Agent({ name: 'Professional Chef', instructions: 'You are a professional chef. Help users create detailed recipes.', model: openai('gpt-4o'), }); // 在路由或服务中发起请求 const response = await chefAgent.generate('如何做一碗正宗的担担面？'); console.log(response.text);

核心机制拆解

1 独立 System Prompt

通过 instructions 直接对智能体设定背景人设与行为准则，限定角色边界。

2 统一的 AI SDK 模型接入

原生整合 `@ai-sdk/openai` 等业内标准，多厂商模型更换仅需一行修改。

3 简洁调用方法

提供 `.generate()` 获取结果，亦支持 `.stream()` 用于前端打字机效果。

PRACTICAL 2 / CUSTOM TOOLS

实战 2：赋能 Agent：自定义并绑定 Tool

1. 定义工具 (tools.ts)
import { createTool } from '@mastra/core'; import { z } from 'zod'; export const getWeather = createTool({ id: 'get-weather', description: 'Get current weather of a city', inputSchema: z.object({ city: z.string().describe('The city name'), }), execute: async ({ context }) => { // 原子异步执行逻辑 return await fetchWeatherApi(context.city); } });
2. 绑定工具 (agent.ts)
import { chefAgent } from './agent'; import { getWeather } from './tools'; export const weatherChefAgent = new Agent({ name: 'Weather Chef', instructions: 'Custom recipes based on current weather.', model: openai('gpt-4o'), tools: { getWeather }, // 一键绑定 }); // Agent 会识别出需要先调用该工具获取天气数据，再作答
基于 Zod 模式架构设计 · 运行时与编译期强安全保障

ADVANCED / WORKFLOWS

进阶之道：用 Workflows 掌控复杂逻辑

[ Step A ] : 用户请求输入 (Trigger)
[ Step B1 ] : LLM 标签提取
[ Step B2 ] : 数据库查询
[ Step C ] : 最终报告聚合及邮件发送

为什么放弃 Agentic Loop 拥抱 Workflow？

1 确定性高，杜绝死循环

当业务执行链路非常严苛（如：退款申请审核），无需让 LLM 盲目猜测 Tool 的调用顺序，而是直接使用链式拓扑控制执行边界。

2 链式与分支控制 API

Mastra 提供了直观的链式逻辑：`.then()` 顺序流转、`.parallel()` 并行控制、以及 `.branch()` 逻辑判断，流程编排极其灵活舒适。

3 生产级故障恢复保障

节点内置完美状态持久化、自动重试机制、调试链路追踪等工程指标，支持人机协同确认机制（Human-in-the-loop）。
