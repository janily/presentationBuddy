WHY MASTRA

为什么是 Mastra？

用开发者真正在意的技术特性，建立对框架的信任。

TypeScript-First
原生类型安全，端到端推导，无需胶水代码。
Local-First
内置 Playground，本地优先开发与实时调试。
生态融合
无缝对接 Vercel AI SDK、Zod 等现代 TS 库。
更轻量
比 LangChain 更贴合 JS/TS 开发者的心智模型。

STEP 01

定义你的基础 Agent

用最少的 TypeScript 代码定义一个基础 Agent。

1
引入核心包
从 @mastra/core 导入 Agent 类。
2
配置模型
通过 @ai-sdk/openai 接入 LLM。
3
定义 Instructions
用自然语言设定 Agent 的角色与边界。
4
导出实例
保存至 src/mastra/agents/index.ts。
src/mastra/agents/index.ts
import { Agent } from '@mastra/core';
import { openai } from '@ai-sdk/openai';

export const weatherAgent = new Agent({
  name: 'weather-agent',
  instructions: `你是一个天气助手， 只使用工具返回的真实数据回答。`,
  model: openai('gpt-4o-mini'),
});

STEP 02

为 Agent 添加 Tools

赋予 Agent 外部调用能力，体验 TS 类型安全的优势。

src/mastra/tools/weather.ts
import { createTool } from '@mastra/core/tools';
import { z } from 'zod';

export const weatherTool = createTool({
  id: 'get-weather',
  description: '获取指定城市天气',
  inputSchema: z.object({
    city: z.string(), // 类型自动推导
  }),
  execute: async ({ context }) => {
    return await fetchWeather(context.city);
  },
});
src/mastra/agents/index.ts
new Agent({
  ...,
  tools: { weatherTool },
});
💡 类型安全的关键
Zod Schema 不仅做运行时校验，还会自动生成给 LLM 的 Function Calling 参数描述 —— context 参数的类型由 inputSchema 完全推导，写错字段名 TS 直接报错。

TROUBLESHOOTING

常见入门陷阱与排查

提前规避 TS 开发者常遇到的运行时错误。

陷阱 1
API Key 未设置
未在 .env 中配置模型密钥导致请求失败。
OPENAI_API_KEY=sk-***  // process.env.OPENAI_API_KEY
陷阱 2
Schema 与 execute 参数类型不匹配
修改 inputSchema 后忘记同步 execute 逻辑，TS 会在编译期直接报错拦截。
调试技巧
开启 Mastra 内置日志，查看 LLM 原始请求 / 响应与 tool-call 全过程，快速定位问题根因。
