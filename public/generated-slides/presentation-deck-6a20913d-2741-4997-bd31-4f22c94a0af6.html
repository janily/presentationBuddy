为什么选择 Mastra？
Eco-System Sync

传统 Python 衍生框架

  • ✗ 生态割裂：在 JS/TS 中写包装包（LangChainJS 等），由于多层封装和转换，API 调用复杂难调试。
  • ✗ 类型丢失：输入/输出链路缺乏保障，复杂场景下的返回值类型难以推导，经常遇到 runtime error。
  • ✗ 运行冗余：强行打包 Python 或复杂解释器，带来了巨大的包体积与不必要的跨语言互操作开销。
VS

Mastra TS 原生优势

  • ✓ 生态契合：无缝融入 Vercel AI SDK、Next.js 与 Node 生态，面向原生 TS 标准设计，极致轻量。
  • ✓ 全链路强类型：完美结合 Zod 运行时校验，从 Tool 输入定义到 Agent 的 Output 结果实现彻底类型安全。
  • ✓ 现代工程化：无多余运行时转译，极快启动，完全契合 Serverless 和 Cloudflare Workers 等轻量边缘计算。
Mastra 框架的三大核心支柱
Core Concepts

智能体 (Agents)

基于大语言模型（LLM）的多轮对话、系统提示词（System Prompt）和自主决策环。作为大脑，它可以自主决定在什么时机调用外部工具。

工具 (Tools)

定义明确输入（Schema）与输出类型的可执行异步函数。完全由 Zod 支持运行时校验，让 Agent 可以安全地调用数据库、第三方 API 或执行本地计算。

工作流 (Workflows)

兼顾分支、并行和确定性的图形化流程编排（DAG）。将多步骤的复杂业务逻辑固化，用确定性的规则来约束 LLM 决策，降低失控风险。

Quick Start

1. 安装依赖

Mastra 核心库非常纯粹，零捆绑包，支持在你的任意 TS/JS 新老项目中直接安装使用。

2. 声明 Agent 实例

通过原生 ESM 语法直接引入 Agent，绑定大模型提供方和系统提示词以指导 AI 核心逻辑行为。

3. 执行提问

极简的 Promise API 式调用。通过调用 .text() 函数并传入输入直接发起第一轮推理和获取答复。

quickstart.ts
TypeScript
// 1. 导入 Mastra Agent 类
import { Agent } from '@mastra/core';

// 2. 创建一个拥有特定人设的 Agent 实例
const codeHelper = new Agent({
  name: 'CodeHelper',
  instructions: '你是一个专业的 TypeScript 开发，回答应当极简高效',
  model: { provider: 'openai', name: 'gpt-4o' },
});

// 3. 运行 Agent 并输出回答
async function main() {
  const response = await codeHelper.text({
    input: '如何实现一个 Zod 到 JSON Schema 的转换？'
  });
  console.log(response.text);
}

main();

search-tool.ts
TypeScript
import { createTool } from '@mastra/core';
import { z } from 'zod';

export const searchDocsTool = createTool({
  id: 'searchDocs',
  description: '搜索 Mastra 框架官方技术文档',
  // 使用 Zod 运行时强校验输入结构
  inputSchema: z.object({
    query: z.string().describe('搜索关键词'),
    limit: z.number().optional().default(5)
  }),
  // 输入参数 query 自动推导为 string 类型
  execute: async ({ query, limit }) => {
    // 执行底层的数据库检索或者搜索引擎 API
    return await fetchRemoteDocs(query, limit);
  }
});

Cognitive Layer

● 会话持久化 (Thread Session)

内置高效的会话追踪。不仅记录多轮对话中的一问一答，还可以动态提取结构化上下文，并在多轮运行中实现自动化摘要存储，减缓 Token 膨胀速度。

● RAG 原生集成 (Vector Retrier)

内置多种向量检索器与 Embeddings 处理层，从文本分块、生成 Vector 到检索相关背景，完全原生对接，无需引入外部臃肿组件。

● MCP 协议原生支持

作为下一代大模型工具互操作协议（Model Context Protocol）的追随者，Mastra 支持无缝对接任何兼容 MCP 的服务器，打破孤岛数据瓶颈。

Real World Practice

1. 抓取与搜集 (Scrape)

使用 FirecrawlTool 抓取主流开源项目的 Releases 或 GitHub 更新日志，剥离 HTML、CSS 等冗余信息，沉淀出纯净的源数据内容。

2. 结构化抽取 (Process)

Agent 接管数据，分析新版本中的 Feature（特性）、Bug Fixes（修复）与 Breaking Changes（破坏性更新），提取版本号与作者信息。

3. 生成报告 (Output)

通过工具将分类归纳好的数据输出为一份排版精美、层级分明的 Markdown 日报，并自动推送至开发者 Slack 频道或 GitHub Wiki 页面。

可观测性与生产部署 (Observability)
Telemetry & Ops

1. 本地可交互 Dev 看板

Mastra 自带本地 Web 调试控制台。在本地运行调试命令时，一键启动本地 GUI，查看每一个 Agent 的多轮对话历史，清晰观察 Agent 具体的调用工具耗时与 Trace Tracing 深度追踪，无需引入第三方昂贵的 SAAS 服务。

2. 生产级 OpenTelemetry 支持

原生整合 OpenTelemetry 标准链路。支持无缝输出标准的 Span 埋点到 Datadog、Jaeger、Arize Phoenix 等应用性能和可观测性监控平台，在关键推理链、中间状态和 LLM 网络请求调用上拥有高可靠保障。
