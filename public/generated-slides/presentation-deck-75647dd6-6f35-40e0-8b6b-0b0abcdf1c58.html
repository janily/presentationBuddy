WHY FRONTEND NEEDS MASTRA?

  • 前端痛点：AI 逻辑常被迫用 Python 写
  • Mastra 优势：TS 原生、与 Node.js 生态无缝、类型安全
  • 受众定位：有 Node.js 基础即可上手
concepts.ascii
+---------+ +------+ | AGENT | ----> | LLM | +---------+ +------+ | | uses v +---------+ | TOOL | +---------+ [Agent] = LLM + Instructions + Tools

MASTRA 核心概念速览

  • Agent：LLM + 系统指令 + 工具绑定的实体
  • Tool：带有 Zod 参数校验的异步函数
  • Workflow：多步骤编排（本教程聚焦 Agent+Tool）

src/tools/weather.ts
import { createTool } from '@mastra/core'; import { z } from 'zod'; export const getWeather = createTool({ name: 'getWeather', description: 'Get weather for a city', parameters: z.object({ city: z.string() }), execute: async ({ city }) => { // call any Node.js API return { city, temp: 22 }; } });

STEP 1: 定义你的第一个 TOOL

  • 示例工具：获取城市天气 getWeather
  • 使用 createTool 定义，用 zod 约束 city 参数
  • execute 函数内可调用任意 Node.js API
src/agents/index.ts
import { createAgent } from '@mastra/core'; import { getWeather } from '../tools/weather'; export const agent = createAgent({ model: 'gpt-4o-mini', instructions: 'You are a weather assistant', tools: [getWeather] });

STEP 2: 创建带工具的 AGENT

  • 使用 createAgent 实例化
  • 配置 model（如 gpt-4o-mini）与 instructions
  • 将上一步的 Tool 数组传入 agent

深入执行流：AGENT-TOOL 通信机制

  • LLM 意图识别：决定是否需要调用 Tool
  • Mastra 运行时：安全执行 Tool 并回传结果
  • 循环收敛：LLM 基于 Tool 结果生成最终响应
stream.ts
for await (const chunk of agent.stream('北京天气？')) { process.stdout.write(chunk.text); } // tokens streamed to UI > 北京当前天气22°C

流式响应：对接前端界面

  • 前端关心点：用 agent.stream() 替代 generate
  • 异步迭代器处理 token 流
  • 轻松接入 React/Vue 的响应式状态
