// TOOLS

声明 Tools：Zod 驱动的强类型安全边界

  • ▸使用 createTool 精确封装：定义唯一的 id 与 description 作为 LLM 识别凭证
  • ▸Zod 定义边界：inputSchema 与 outputSchema 强制约束数据流，杜绝模型幻觉导致的入参错误
  • ▸强大的 TS 类型推断：在 execute 执行体中完美享受 IDE 自动补全与类型检查
weather-tool.ts
import { createTool } from '@mastra/core'; import { z } from 'zod'; export const weatherTool = createTool({ id: 'get-weather', description: '查询指定城市实时天气', inputSchema: z.object({ city: z.string(), }), outputSchema: z.object({ temperature: z.number(), }), execute: async ({ input }) => { return fetchWeather(input.city); }, });
