Agents

强类型 Tools：基于 Zod 的安全接口设计

Declarative schema validation and flawless execution loops

src/mastra/tools/weather.ts

import { createTool } from '@mastra/core';

import { z } from 'zod';


export const weatherTool = createTool({

  id: 'get-weather',

  description: '获取指定城市的实时天气数据',

  inputSchema: z.object({

    city: z.string().describe('城市名称，如: 北京'),

    unit: z.enum(['C', 'F']).optional()

  }),

  execute: async ({ input, context }) => {

    // 此时 input 经过 Zod 强校验，天然具备严格 IDE 自动推导提示

    const { city, unit = 'C' } = input;

    const data = await fetchWeather(city);

    return { temp: data.temp, unit };

  }

});

工具契约运行时双重屏障

AI 智能体产生的大部分运行时错误源于“对参数的野蛮理解”（例如模型生成非标准格式、字符串包裹错误）。Mastra 借助 Zod 在执行前筑起第一道城墙。

01
LLM 结构化参数输出
LLM 读取描述，自动生成 JSON 输入。
02
Zod 强约束动态校验 Type Safe ✓
严格限制类型范围，错误调用自动阻断或重试。
03
Node.js 高性能执行逻辑
安全进入 execute，直接获取带类型保障的参数值。
★ 自定义 Tracing 系统将自动记录每一次 Tool 的输入与耗时评估。

Agent 声明式开发：让 LLM 获得自主决策力

Assemble agents with models, system guidelines, and safe tools

src/mastra/agents/index.ts

import { Agent } from '@mastra/core';

import { weatherTool } from '../tools/weather';


export const weatherAgent = new Agent({

  name: 'Weather Agent',

  instructions: '你是一个聪明的天气管家。' +

                '请务必使用天气工具查询实时温度后，' +

                '给用户提供最贴心的穿衣与出行规划建议。',

  model: {

    provider: 'OPENAI',

    name: 'gpt-4o',

    toolChoice: 'auto',

  },

  tools: {

    getWeather: weatherTool

  }

});

走向生产环境：Mastra 进阶生态与下一步

Ready to deploy? Discover core enterprise extensions

内置可观测性 (Observability)

开箱即用的调用链追踪 (Traces) 与评估 (Evals)。内置 OpenTelemetry 支持，可无缝对接 Phoenix, Honeycomb 等第三方平台。

MCP 原生模型上下文协议

深度集成 Model Context Protocol (MCP) 服务器，支持一键直接共享、接入外部丰富的工具和数据生态环境。

内置向量检索 (RAG)

极其平滑的数据入库切片（Chunking）机制，内置本地 LibSQL / SQLite，且全面兼容 Pinecone, PGVector 进阶接入。

一键多云轻松部署

可作为独立 Express / Fastify 微服务运行，也可以零摩擦地打包成 serverless 边缘路由（Edge API Router）。
