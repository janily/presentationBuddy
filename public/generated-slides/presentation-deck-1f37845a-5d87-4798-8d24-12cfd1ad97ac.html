Mastra 核心概念速览

+---------+       +-------+
| [Agent] | ----> | [LLM] |
+---------+       +-------+
     |
     v
+---------+
| [Tool]  |  (Zod 校验的异步函数)
+---------+

Agent = LLM + 系统指令 + 工具绑定
  • Agent：LLM + 系统指令 + 工具绑定的实体
  • Tool：带有 Zod 参数校验的异步函数
  • Workflow：多步骤编排（本教程聚焦 Agent+Tool）

Step 1: 定义你的第一个 Tool

src/tools/weather.ts 
import { createTool } from '@mastra/core';
import { z } from 'zod';

export const getWeather = createTool({
  id: 'getWeather',
  description: '获取指定城市的当前天气',
  inputSchema: z.object({
    city: z.string().describe('城市名称'),
  }),
  execute: async ({ context }) => {
    const { city } = context;
    // 可调用任意 Node.js API
    const res = await fetch(`https://api.weather.com/${city}`);
    const data = await res.json();
    return { city, temp: data.temp, condition: data.condition };
  },
});
  • 示例工具：获取城市天气 getWeather
  • 使用 createTool 定义，用 zod 约束 city 参数
  • execute 函数内可调用任意 Node.js API
  • Zod 提供运行时类型校验与文档

Step 2: 创建带工具的 Agent

src/agents/weather.ts 
import { createAgent } from '@mastra/core';
import { getWeather } from '../tools/weather';

export const weatherAgent = createAgent({
  id: 'weather-agent',
  model: 'gpt-4o-mini',
  instructions: '你是一个天气助手，使用工具回答天气问题。',
  tools: [getWeather], // 绑定工具数组
});
  • 使用 createAgent 实例化
  • 配置 model（如 gpt-4o-mini）与 instructions
  • 将上一步的 Tool 数组传入 agent
  • 黄色框标出 tools: [weatherTool] 绑定动作

Step 3: 运行你的 Agent（可运行示例）

src/index.ts

深入执行流：Agent-Tool 通信机制

User  --> Agent  --> LLM
                   |
                   v
                 Tool
                   |
                   v
                  LLM  --> Agent  --> User
(绿色虚线表示循环收敛)
  • LLM 意图识别：决定是否需要调用 Tool
  • Mastra 运行时：安全执行 Tool 并回传结果
  • 循环收敛：LLM 基于 Tool 结果生成最终响应

流式响应：对接前端界面

frontend-stream.ts 
const stream = await weatherAgent.stream('上海天气?');
for await (const chunk of stream) {
  process.stdout.write(chunk.text);
}
// 输出片段: 上海当前... 晴 ... 25°C
// 异步迭代器轻松接入 React/Vue 响应式状态
  • 前端关心点：用 agent.stream() 替代 generate
  • 异步迭代器处理 token 流
  • 轻松接入 React/Vue 的响应式状态
