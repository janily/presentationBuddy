Architecture

Mastra 的三大支柱：Agents, Tools & Workflows

🤖

Agents 智能体

集成 LLM 决策、System Prompt 设置与内置长短期记忆（Memory）的控制中枢。

Key capabilities:

角色扮演、内置上下文记忆、多模型自动适配切换

⚙️

Tools 工具集

通过强类型标准定义的外部执行函数。利用 TypeScript 类型守卫和 Zod Schema 赋予大模型精准的数据交互能力。

Key capabilities:

Zod Schema 强制入参校验、自动生成 OpenAPI 文档、一键接入 SDK

🔄

Workflows 工作流

基于有向无环图（DAG）的复杂状态机编排引擎。确保大型非确定性 AI 任务能够按设定的商业逻辑稳定运行。

Key capabilities:

强类型变量流转、带条件逻辑的控制流分支、重试与异常容错

Implementation

实战：使用 createTool 与 Zod 构建首个 Agent

src/mastra/tools/weather.ts
import { createTool } from '@mastra/core';
import { z } from 'zod';

export const weatherTool = createTool({
  id: 'get-weather',
  description: 'Fetch current weather condition for location',
  inputSchema: z.object({
    location: z.string().describe('The city name'),
    unit: z.enum(['c', 'f']).optional(),
  }),
  execute: async ({ input }) => {
    // Input validated automatically by Zod!
    return { temp: 24, location: input.location };
  }
});

强类型参数校验与自愈能力

利用 Zod 作为输入模式，Mastra 能够在大模型发送错误格式入参时，自动抛出高可读性校验错误并辅助模型自动重试，确保程序绝不崩溃。

src/mastra/agents/index.ts
import { Agent } from '@mastra/core';
import { weatherTool } from '../tools/weather';

export const weatherAgent = new Agent({
  name: 'Weather Agent',
  instructions: 'You are a helpful assistant',
  model: { provider: 'openai', name: 'gpt-4o' },
  tools: { weatherTool }
});
Orchestration

进阶：使用 Workflows 编排复杂的多步骤流程

AI 驱动的有向无环图 (DAG) 状态转移

1 Fetch User Data
2 LLM Analysis Step
Condition: Score > 8
Auto Approve
Condition: Manual Review
Trigger Notification

声明式链式图定义

使用纯 TypeScript，你可以通过极简的链式调用，声明节点（`.step()`）与连线转移逻辑（`.then()`）。在开发期间完全保证步骤间数据输入和输出的类型对齐。

src/mastra/workflows/index.ts
import { Workflow } from '@mastra/core';

export const onboardingWorkflow = new Workflow({
  name: 'user-onboarding'
})
  .step(fetchUserDataStep)
  .then(analyzeProfileStep)
  .after(analyzeProfileStep)
  .transition({
    condition: (data) => data.score > 8,
    step: autoApproveStep
  });
