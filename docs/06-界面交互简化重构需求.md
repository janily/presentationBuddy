# 界面交互简化重构需求：极简双栏 · 聊天驱动 · 全中文界面

**文档版本**: v1.0
**创建日期**: 2026-07-08
**优先级**: P0（交互重构）+ P1（死代码清理）
**背景**: `01~05` 号需求文档实施后，产品功能链路（对话 → 大纲 → 生成 HTML → 预览）已经跑通，但产品负责人验收时认为**整体界面仍然复杂冗余，不够简洁**。目标是对齐参考设计（类 Anthropic 风格的极简双栏）：一个"干活的演示文稿助手"——右侧一个干净的聊天面板，左侧大面积实时预览画布，仅此而已。本文档基于当前代码库的实际实现定位冗余点，给出完整改造方案。

**范围声明**：本次为**纯前端交互重构**。`/api/agent-chat`、`/api/analyze`、`/api/preview` 路由逻辑、Mastra workflow 及三个 presentation agent、`use-presentation-workflow.ts` 的数据契约、suspend/resume 审批机制、watchdog 超时机制一律**不改动**（仅文案中文化）。唯一的后端文件删除是 interior 遗留的 `src/app/api/upload/route.ts`，属死代码清理。

## 二、目标信息架构

### 2.1 设计原则

1. **全屏两栏，没有页面级 header**：左侧画布 + 右侧聊天面板，`h-screen` 内各自滚动，整页不滚动。
2. **唯一信息源**：任何一份信息（进度、大纲、下载入口）在全页只出现一处。
3. **左侧管"内容"，右侧管"对话与决策"**。
4. **右侧面板有且只有一个 header**，一行式。

### 2.2 布局示意

```
┌────────────────────────────────────────────┬──────────────────────────┐
│                                            │ ✦ Presentation Buddy   ↺ │ ← 面板唯一 header
│                                            ├──────────────────────────┤
│              左：画布 / 预览区                │                          │
│                                            │      消息流 或 空态        │
│    （空态 / 大纲流式 / 生成进度 / iframe）      │                          │
│                                            ├──────────────────────────┤
│                                            │ [建议 chips（分阶段可选）]  │
│                                            │ ┌────────────────────┐[➤]│
│                                            │ └────────────────────┘   │
│                                            │  Enter 发送 · Shift+En…  │
└────────────────────────────────────────────┴──────────────────────────┘
```

### 2.3 职责划分

| 区域 | 承担 | 明确不承担 |
|------|------|-----------|
| 左侧画布 | 一切"内容与进度"：空态引导、大纲流式渲染、生成进度条、HTML iframe 预览、**唯一下载入口** | 不放对话、不放确认按钮 |
| 右侧面板 | 一切"对话与决策"：聊天、大纲确认按钮、错误重试、生成中新请求的排队/重启决策 | **不放进度条、不放大纲内容、不放下载按钮** |

### 2.4 具体布局改动（`presentation-workspace.tsx`）

1. **删除整个页面级 `<header>`**（h1 + 副标题 + Start over 按钮）。品牌名下沉到右侧面板 header。
2. **删除 aside 外层标题块**（第 33-39 行的 "Agent" 块）。`<aside>` 退化为纯定位容器，直接渲染 AgentPanel，消除双层圆角/边框/内边距嵌套。
3. **Start over 改为右侧面板 header 最右侧的图标按钮**（`RotateCcw` 图标，`aria-label="重新开始"`，加 `title` 提示），对应参考设计中面板 header 的"关闭位"。`onStartOver` 回调透传给 AgentPanel。
4. 根容器改为 `h-screen` 两栏 grid：左 `minmax(0,1fr)`，右约 `400px`，左右各自 `overflow-y-auto`。现有的 `max-w-[1600px]` 居中容器与 `min-h-[calc(100vh-…)]` 类魔法数随 header 一起删除。

## 三、右侧聊天面板重构（`agent-panel.tsx`）

### 3.1 面板 header（唯一一层）

一行式：`✦ 图标 + "Presentation Buddy" +（右侧）重新开始图标按钮`。

**删除**：内层 h3 "Presentation agent"、phase 徽章（`getPhaseLabel` 的徽章渲染）、随 phase 变化的 subtitle。阶段信息已由左侧画布和消息流传达，无需徽章。

### 3.2 空态设计（对齐参考设计）

**删除 `initialGuidanceMessage`**（`presentation-studio.tsx` 中预置的第一条 assistant 消息），`chatMessages` 初始改为 `[]`。当 phase 为 `briefing` 且消息列表为空时，消息区显示空态（不进 messages 数组）：

- 居中小图标（Sparkles，柔和底色圆形/圆角块）
- 主文案："**想做什么演示文稿？**"
- 副文案："告诉我主题、受众和大致页数，我会先生成大纲给你确认。"
- 建议问题**垂直列表**（3 条，整行可点，点击即作为用户消息发送）：
  1. 帮我做一份产品发布会的演示文稿
  2. 为投资人准备一份 10 页的融资路演
  3. 给团队做一份季度复盘汇报

### 3.3 消息流卡片去留

| 卡片 | 处理 | 说明 |
|------|------|------|
| 文本气泡（user / assistant） | **保留** | 样式不变 |
| `OutlineReviewCard` | **简化保留** | 大纲确认是核心决策点。压缩为轻卡片：标题"大纲已就绪（N 页）" + 说明"确认后开始生成，或直接在下方告诉我修改意见。" + **单个「生成演示文稿」按钮**。**删除 "Edit outline" 按钮及 `onOpenOutline` prop**。`disabledReason` 逻辑保留（文案中文化） |
| `ProgressCard` | **删除** | 进度唯一信息源在左侧。`outlining`/`generating` 阶段仅在消息流底部显示一行**轻量状态行**（复用 TypingDots 样式 + 文字"正在生成大纲…"/"正在生成演示文稿…"），由 `phase` 直接派生渲染，不进 messages 数组，不带进度百分比 |
| `CompleteCard` | **降级为普通文本气泡** | 保留 `complete` 消息 kind，仅把渲染函数换成文本气泡（改动最小）：文案"已生成 N 页演示文稿，预览在左侧。想调整哪里，直接告诉我。"**删除卡内"下载"与"View outline"按钮**（下载只留左侧 `HtmlPreview` 工具栏） |
| `ErrorCard` | **保留** | 仍是错误重试的唯一入口，文案中文化 |
| `GenerationRequestCard` | **保留** | "生成中收到新请求 → 完成后应用 / 停止并重新开始"是必要的功能决策卡，仅文案中文化 |

`AgentMessage` 类型相应收窄：删除 `progress` 变体（`systemMessages` 派生逻辑中同步删除 outlining/generating 的 progress 消息分支）。

### 3.4 建议 chips 与输入区

- **briefing**：chips 由空态垂直列表承担，输入框上方不再放 chips。
- **reviewing**：保留 chips（输入框上方）：「增加一页客户案例」「语气更正式一些」「删掉第 3 页」。
- **previewing**：保留 chips：「整体更简洁」「换一种视觉风格」「修改配色」。
- **outlining / generating / error**：无 chips。
- 输入框：保留 textarea + 发送按钮；**表单下方新增一行灰色小字："Enter 发送 · Shift+Enter 换行"**（参考设计的落款行）。placeholder 按 phase 中文化（见第五节）。

## 七、分期实施步骤与验收标准

### P0：交互重构（先做，效果立即可见）

**步骤 1 — 布局收敛**（`presentation-workspace.tsx`，必要时 `globals.css`）

删页面 header、删 aside 外层标题块，改 `h-screen` 两栏各自滚动，`onStartOver` 透传给 AgentPanel。

> 验收：整页没有外层标题；右侧只有一个面板 header；「重新开始」按钮可用且能重置全部状态。

**步骤 2 — 聊天面板重构**（`agent-panel.tsx`）

新一行式 header（品牌 + 重开按钮）；空态组件；删 `ProgressCard`；简化 `OutlineReviewCard`（删 Edit outline / `onOpenOutline` prop）；`complete` 渲染为文本气泡；新增状态行与 "Enter 发送" 提示行；`phaseCopy` 全中文。

> 验收：briefing 无消息时显示空态三条建议问题，点击即发送；reviewing 阶段只有一张确认卡一个按钮；generating 时右侧无进度条，只有一行状态文字。

**步骤 3 — 预览区中文化与收敛**（`presentation-preview-pane.tsx`）

空态简化；`OutlineProgress` / `GenerationProgress` 中文化；review 态标题中文；「下载 HTML」为全站唯一下载入口。
